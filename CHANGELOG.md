## [0.4.0] - 2026-03-14

### ✨ 新增 (Added)

#### 总控池主动话题 (Global Pool)

- **功能描述**：在白名单聊天流中汇总近期消息，自动构建候选话题，并由 LLM 决定是否主动发起。
- **核心机制**：
  - 白名单聊天流消息采集与最小化存储
  - 基于兴趣度、新颖度、跨流热度提升、重复惩罚的候选话题构建
  - 支持全局冷却、单流冷却、每日额度、屏蔽关键词与策略档位控制
- **配置项**：`[global_pool]`，支持 `enabled`, `whitelist_streams`, `tick_interval_seconds`, `lookback_hours`, `stream_policy`, `policy_profiles` 等

#### 总控池管理命令

- **新增命令**：`/amind_pool`
- **支持子命令**：
  - `status` - 查看总控池运行状态
  - `dryrun` - 查看当前候选话题与决策结果
  - `whitelist` - 查看白名单聊天流
  - `profile` - 查看各聊天流策略映射

#### 聊天流隔离状态

- **功能描述**：话题新增 `stream_ids` 绑定能力，并为每个聊天流维护独立状态记录。
- **实现机制**：
  - 数据库新增 `amind_topic_stream_states` 表
  - 话题可按聊天流独立统计活跃度、检查次数、自发起次数等状态
  - 为总控池主动发起提供更清晰的流级隔离基础

### 🔧 变更 (Changed)

#### 总控池启动方式

- **变更前**：插件仅启动 Plan 相关周期任务。
- **变更后**：插件启动时会统一注册总控池周期任务，功能是否实际执行由 `global_pool.enabled` 控制。
- **影响**：总控池功能可随配置即时启停，启动链路更一致。

### 🐛 修复 (Fixed)

#### Plan 启动判定

- **修复**：修复了在 `plugin_config` 路径下，`plan1.enabled = false` 仍可能被识别为启用的问题。
- **效果**：禁用的 Plan 不会再误启动后台 Tick 任务。

#### 核心包循环导入

- **修复**：将 `core` 包调整为惰性导出，避免独立导入 `database.py` 时触发循环导入。
- **效果**：数据库模块与插件入口的导入路径更稳定，测试与运行环境一致性更好。

### 📝 文档 (Documentation)

- 更新 `_manifest.json` 版本至 0.4.0
- 更新 `README.md` 版本徽章与当前版本说明
- 更新 `CHANGELOG.md` 补充总控池与流级状态隔离说明

## [0.3.0] - 2026-02-02

### ✨ 新增 (Added)

#### 话题捕捉 (Topic Capture)

- **功能描述**：Bot 像真人一样“潜水”观察群聊，基于 LLM 上下文分析，在合适的时机主动介入。
- **核心机制**：
  - 独立于自动发起的概率检测机制
  - **LLM 裁判**：基于信息缺失、被忽视提问、情绪共鸣、观点僵局、话题早夭 5 大维度判断
  - **上下文感知**：读取最近 20 条消息进行分析
- **配置项**：`[plan1.topic_capture]`，支持 `enabled`, `probability`, `interval`, `min_messages`

#### 多 API Key 轮询

- **功能描述**：支持配置多个 Tavily API Key，自动处理额度耗尽或请求限制
- **实现机制**：如果 Key A 报错 (401/403/429)，自动切换 Key B 重试
- **配置变更**：`tavily_api_key` 现支持字符串列表 `["key1", "key2"]`

#### 高级话题策略

- **Epsilon-Greedy 探索**：以一定概率 (`exploration_epsilon`) 忽略历史偏好，强制通过搜索发现新话题
- **动态查询生成**：LLM 自动将宽泛关键词（如“科技”）转化为具体时效性查询（如“2026年2月AI技术突破”），极大提升搜索相关性
- **统一创作管线**：重构了话题创建流程，确保所有自动话题都经过完整的 [搜索->头脑风暴->决策] 下一代流程

### 🐛 修复 (Fixed)

#### 启动延迟与持久化

- **修复**：修复了“上次执行时间”仅在成功时保存的 Bug。现在无论成功失败都会持久化时间戳。
- **效果**：确保了重启后不会立即刷屏，且冷却时间严格生效，防止失败后的高频重试 (Spam)。

### 📝 文档 (Documentation)

- 更新 `_manifest.json` 版本至 0.3.0
- 更新 `README.md` 添加新功能配置说明

## [0.2.0] - 2025-01-13

### ✨ 新增 (Added)

#### 关键词权重管理系统

- **功能描述**：支持手动配置不同类别关键词的权重，精确控制话题生成倾向
- **配置项**：`[planX.keyword_weights]` 配置节
  - `enable_manual_weights` - 启用手动权重（覆盖AI自动分析）
  - `tech_weight` - 技术类关键词权重（0.0-1.0）
  - `science_weight` - 科学类关键词权重（0.0-1.0）
  - `social_weight` - 社会类关键词权重（0.0-1.0）
  - `entertainment_weight` - 娱乐类关键词权重（0.0-1.0）
- **新增命令**：
  - `/kw show [plan]` - 查看当前权重配置
  - `/kw set [plan] <参数>` - 设置权重
  - `/kw enable [plan]` - 启用手动权重
  - `/kw disable [plan]` - 启用自动偏好
  - `/kw reset [plan]` - 重置为默认值

#### 关键词轮询机制

- **功能描述**：优化关键词选择逻辑，确保所有配置的关键词都能被均匀使用
- **实现方式**：
  - 添加类变量 `_keyword_indices` 跟踪每个Plan的关键词索引
  - 每次触发自动移动索引位置（每次移动3位）
  - 支持多Plan独立轮询
- **效果**：之前只使用前3个关键词的Bug已修复，现在所有关键词都会被使用

#### 日志系统优化

- **功能描述**：统一使用logger系统，所有日志可通过配置灵活控制
- **修改文件**：
  - `core/a_mind_plan_tick_task.py` - 20个print改为logger
  - `handlers/a_mind_start_handler.py` - 5个调试print改为logger
  - `database.py` - 3个数据库print改为logger
  - `core/amind_logger.py` - 移除配置调试print
- **日志级别分类**：
  - **DEBUG** - 详细调试信息（Tick执行、配置读取、概率检查等）
  - **INFO** - 重要事件（概率检查通过、自动发起成功等）
  - **WARNING** - 警告信息（并发检查、配置为空等）
  - **ERROR** - 错误信息（队列处理失败、数据库错误等）
- **新增配置**：
  - `[logging.preset]` - 预设模式（minimal/normal/verbose/debug）
  - `[logging.level]` - 全局级别覆盖
  - `[logging.modules]` - 模块级别独立控制

### 🔧 变更 (Changed)

#### 关键词选择逻辑

- **变更前**：每次只使用前3个关键词，后面的永远不会被使用
- **变更后**：通过轮询机制确保所有关键词均匀使用
- **影响**：话题类型更加多样化，避免话题单调

#### 日志输出方式

- **变更前**：大量使用print()直接输出，无法控制
- **变更后**：统一使用logger系统，支持灵活配置
- **影响**：可以通过配置控制日志详细程度，减少无关输出

### 🐛 修复 (Fixed)

#### 关键词池Bug

- **问题**：关键词配置18个，但只有前3个被使用
- **原因**：`search_queries[:3]` 固定取前3个
- **修复**：实现轮询机制，每次取不同位置的关键词
- **影响**：现在所有配置的关键词都能被使用

#### 日志混乱问题

- **问题**：日志输出混乱，无法控制详细程度
- **原因**：print()输出不受logger系统控制
- **修复**：将所有print改为logger调用
- **影响**：日志更加规范，可通过配置灵活控制
